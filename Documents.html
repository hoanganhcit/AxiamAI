Getting Started

Create a project in the Axiam Console, grab your API keys, and install the SDK for your platform.

Create a Project
In Console → Projects → New. Copy the AXIAM_PROJECT_ID and AXIAM_API_KEY.
Link Your App
Add allowed origins/callbacks for web or package IDs for mobile. Enable Facial Sign‑In and Liveness.
Configure Webhooks
Receive events like user.enrolled, auth.success, auth.challenge. 
// 1) Install
npm i @axiam/web

// 2) Initialize
import { Axiam } from '@axiam/web'
const axiam = new Axiam({ projectId: process.env.NEXT_PUBLIC_AXIAM_PROJECT_ID })

// 3) Enroll + Sign‑In
await axiam.enroll({ userId: 'user_123' })
const session = await axiam.signIn({ policy: 'face_only' })
console.log('session', session.token)

// Swift Package Manager → AxiamSDK
import AxiamSDK
let client = Axiam(projectId: "")
try await client.enroll(userId: "user_123")
let session = try await client.signIn(policy: .faceOnly)
print(session.token)

// Gradle: implementation("com.axiam:android:")
val axiam = Axiam(projectId = AXIAM_PROJECT_ID)
axiam.enroll(userId = "user_123")
val session = axiam.signIn(policy = FaceOnly)
println(session.token)

Facial Sign‑In API

Core endpoints for enrollment, liveness, and authentication.

POST /v1/users/:id/enroll
curl -X POST https://api.axiam.io/v1/users/user_123/enroll \
  -H "Authorization: Bearer $AXIAM_API_KEY" \
  -F video=@selfie.mov
Creates biometric template with active liveness checks.
POST /v1/auth/face
curl -X POST https://api.axiam.io/v1/auth/face \
  -H "Authorization: Bearer $AXIAM_API_KEY" \
  -F video=@challenge.mov
Returns session token on success; supports policy=face_only or step_up.

Security & Compliance

Architecture
AES‑256 at rest, TLS 1.2+ in transit, liveness ML, template encryption via Facelock, optional on‑device capture.
Data & Privacy
Regional data residency (CA/EU/US), configurable retention, per‑user data export/delete APIs, audit logs.

API Reference

REST endpoints with JSON responses. Webhooks sign with HMAC‑SHA256.

POST /v1/tokens/verify
curl -X POST https://api.axiam.io/v1/tokens/verify \
  -H "Authorization: Bearer $AXIAM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"token":""}'

About this simulation

  • Not real biometrics. No face recognition, no identity proofing, no liveness detection. For UX demos only.
  • Local only Captures stay in browser memory; optional hash persistence uses localStorage.
  • Perceptual hash. We compute a 256‑bit aHash and compare using Hamming distance with an adjustable threshold.
  • Suggested next step. Replace the aHash and toy liveness with a real provider’s SDK (e.g., liveness + 3D face map).
